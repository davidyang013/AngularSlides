Data-binding is Awesome!

You don't care about the DOM manipulation any more

<!DOCTYPE html>
<html lang="en" ng-app="demo">
<head>
    <meta charset="UTF-8">
</head>
<body ng-controller="DemoController">
    <input type="text" ng-model="userName">
    <span>{{ userName }}</span>
</body>
</html>
<script type="text/javascript" src="angular.min.js"></script>
<script type="text/javascript" src="demo.js"></script>

(function (angular) {
    'use strict';
    var demo = angular.module('demo', []);
    demo.controller('DemoController', ['$scope', function ($scope) {
        $scope.userName = '';
    }]);
}(angular));

When should we use $apply()


$apply() is used to execute an expression in angular from outside of the angular framework. (For example from browser DOM events, setTimeout, XHR or third party libraries)

How ng-repeat works

The ngRepeat directive instantiates a template once per item from a collection. 

<!DOCTYPE html>
<html lang="en" ng-app="demo">
<head>
    <meta charset="UTF-8">
</head>
<body ng-controller="DemoController">
    <ul>
        <li ng-repeat="item in items">
            {{ item }}
        </li>
    </ul>
</body>
</html>
<script type="text/javascript" src="angular.min.js"></script>
<script type="text/javascript" src="demo.js"></script>

(function (angular) {
    'use strict';
    var demo = angular.module('demo', []);
    demo.controller('DemoController', ['$scope', function ($scope) {
        $scope.items = ['nanfeng', 'beifeng', 'dongfeng'];
    }]);
}(angular));

Lifecycle in AngularJS

Angular initializes automatically upon DOMContentLoaded event or when the angular.js script is evaluated if at that time document.readyState is set to 'complete'. At this point Angular looks for the ng-app directive which designates your application root. If the ng-app directive is found then Angular will:

  • load the module associated with the directive.
  • create the application injector
  • compile the DOM treating the ng-app directive as the root of the compilation. This allows you to tell it to treat only a portion of the DOM as an Angular application.

Differences in DI system between AngularJS and RequireJS

  • AngularJS DI is more like injecting instances without any physical namespace, which means all the angular modules have to be injected before the application bootstrapped
  • RequireJS DI is more like injecting classes or constructors, and you may already know how it workds
